Orchestrator

A lightweight bash orchestrator that routes tasks to autonomous coding agents, with delegation, scheduled jobs, and GitHub sync.

Add a task, the orchestrator picks the best agent, and it gets done. Agents run with full tool access — they read files, edit code, run commands, and report results. Complex tasks get broken into subtasks automatically.

bash yaml yq claude codex opencode

Quick Start

Install via Homebrew

brew tap gabrielkoerich/orchestrator
brew install orchestrator

This installs all dependencies (yq, jq, just, python3) automatically. You still need at least one agent CLI:

brew install --cask claude-code   # Claude
brew install --cask codex         # Codex
brew install opencode             # OpenCode

Optional: brew install gh && gh auth login for GitHub sync.

Background service

brew services start orchestrator

Alternative: install from source

git clone https://github.com/gabrielkoerich/orchestrator.git
cd orchestrator
just install

Usage

# Initialize a project
cd ~/projects/my-app
orchestrator init

# Add a task and let it run
orchestrator add "Fix auth bug" "Login returns 500 on expired tokens" "backend"
orchestrator next

# For complex tasks, break them down into subtasks first
orchestrator plan "Add user auth" "JWT login, signup, password reset" "backend"
orchestrator next

# Or start the server and let it handle everything
orchestrator serve

How It Works

# 1. You add a task
orchestrator add "Fix auth bug" "Login returns 500 on expired tokens" "backend"

# 2. The orchestrator routes it to the best agent
orchestrator route
#    => picks codex, model gpt-4.1, profile: "backend specialist"

# 3. The agent runs in your project directory with full tool access
orchestrator run
#    => reads files, finds the bug, edits code, runs tests

# 4. Done. Results are logged, synced to GitHub, and the next task starts.
orchestrator status
#    ID  STATUS  AGENT  TITLE
#    1   done    codex  Fix auth bug

For complex tasks, the orchestrator breaks them down first:

# Add a complex task with the "plan" label
orchestrator plan "Implement payments" "Stripe integration with webhooks" "backend"

# The orchestrator analyzes the codebase and creates subtasks:
#   => "Add Stripe SDK and config"
#   => "Create payment intent endpoint"
#   => "Add webhook handler"
#   => "Add payment integration tests"
#
# Each subtask is routed to the best agent and runs in parallel.
# The parent task resumes when all children are done.

Features

  • Agentic Mode

    Agents run with full tool access. They read files, edit code, run commands, and verify their own work inside your project directory.

  • LLM Routing

    A cheap/fast LLM classifies each task and picks the best agent, model, and generates a specialized profile with skills and constraints.

  • Context Enrichment

    Agents receive repo file tree, project docs (CLAUDE.md, AGENTS.md, README.md), skill docs, parent/sibling context, git diff, and prior run logs.

  • Task Decomposition

    Complex tasks are broken into subtasks automatically or via orchestrator plan. Each subtask is routed and executed independently.

  • Scheduled Jobs

    Define recurring tasks with cron expressions. Per-job dedup ensures no duplicates while tasks are in-flight.

  • GitHub Sync

    Two-way sync with GitHub Issues and Projects v2. Status labels, comments with progress, and owner tagging when stuck.

  • Error Handling

    Failed tasks are blocked, errors are posted as GitHub issue comments with a blocked label. Auth errors, timeouts, and crashes are classified automatically.

  • Per-Project Config

    Drop a .orchestrator.yml in any project to override GitHub repo, workflow settings, and router config per project.

Commands

orchestrator add "title" "body" "labels"   # Add a task
orchestrator plan "title" "body" "labels"  # Add + decompose into subtasks
orchestrator list                          # List all tasks
orchestrator status                        # Status overview
orchestrator next                          # Route + run next task
orchestrator serve                         # Start background server
orchestrator jobs-add "0 9 * * *" "title"  # Schedule a recurring task
orchestrator init                          # Initialize project
orchestrator projects                      # List all projects
brew services start orchestrator            # macOS auto-start

Scheduled Jobs

Define recurring tasks with cron expressions. Each job creates tasks on schedule, with per-job dedup to prevent duplicates while tasks are in-flight.

# Add a scheduled job
orchestrator jobs-add "0 9 * * *" "Daily sync" "Pull and check" "sync"

# List jobs
orchestrator jobs-list

# Manage jobs
orchestrator jobs-enable daily-sync
orchestrator jobs-disable daily-sync
orchestrator jobs-remove daily-sync

Jobs run automatically when the server is running, or install a crontab entry with orchestrator jobs-install.
